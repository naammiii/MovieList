SQL (Structured Query Language) es un lenguaje de consulta estructurado. Este lenguaje de programación te permite manipular y descargar datos de una base de datos y además tiene capacidad de hacer calculos avanzados y álgebra. Es el lenguaje de  programación para bases de datos relacionales más usado y es utilizado en la mayoría de empresas para el almacenamiento de sus datos.
Una base de datos SQL está formada por tablas con filas y columnas. Las filas son los diferentes registros de la tabla, mientras que las columnas son los campos de información. Entre las carácteristicas más destacables es poder establecer, modificar o eliminar esquemas de relación entre las tablas. Además, permite realizar consultas con distintos parámetros de busqueda y también permite establecer restricciones de integridad a la información que se almacena en la base de datos.
Existen muchos tipos de bases de datos SQL, pero muchas de ellas son muy parecidas y la lógica es la misma. Entre los más populares podemos encontrar MySQL, el cual es el principal gestor de base de datos SQL. MySQL destaca por ser de código abierto y está diseñada para satisfacer las necesidades de todo tipo de usuarios, mientras que Oracle está más orientada al ámbito empresarial.
Oracle fue la primera empresa que comercializó el primer sistema de gestión de bases de datos relacionales y esta consiste en un sistema cerrado que es conocida por ser una de las herramientas más potentes del mercado
El investigador de IBM Edgar F.Codd en 1969 definió el modelo de base de datos relacional. Este modelo se basa en la asociación de claves con varios datos. Unos años más tarde, IBM creó un lenguaje para los sistemas de gesión de bases de datos relacionales basado en el trabajo de Edgar F. Codd. Al principio se llamó SEQUEL (Structured English Query Language) y al cabo de un tiempo acabó llamandose como lo conocemos hoy en dia. Las pruebas conenzaron en 1978 y, mientras tanto, IBM empezó a desarrollar productos comerciales, como SQL/DS en 1981 y DB2 en 1983. Posteriormente otros proveedores, comomo Sybase, Ingres u Oracle, publicaron y comercializando sus propios gestores.
Tambien existen una amplia clase de sistemas de gestión que usan bases de datos NO relacionales, comunmente conocidos como NoSQL. He decidio usar una base de datos relacional frente a la no relacional es porque, a parte de mi experiencia de uso y conocimientos, las relacionales suelen ser mucho más compatibles y, al ser el más extendido y adaptado, obtiene por parte de la comunidad de desarrolladores un mayor soporte.
